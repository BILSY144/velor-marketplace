## Backlog (prioritized, top = next)

1. **[HIGH] Add page-level metadata to the seller-recruitment key pages.** `/apply`, `/sell`, and `/founding` (also `/shop` and `/help`, lower priority) are all `'use client'` page components, so none can export Next.js `metadata` directly -- every one of them currently falls back to the root `app/layout.tsx` title ("Velor - Global Marketplace") and description ("Discover unique products from sellers around the world.") verified live via WebFetch on 2026-07-12. That means search results, social shares (OG/Twitter cards), and AI answer engines see identical, generic copy for the homepage, the seller application page, the founding-seller programme, and the pitch page. Fix: add a server-component `layout.tsx` in each route folder (e.g. `app/apply/layout.tsx`) that only exports `metadata` and renders `{children}` -- this is the standard Next.js App Router pattern for giving a client page.tsx unique metadata, and it does not touch the existing client component at all (zero functional risk). Do `/apply` and `/founding` first (highest seller-recruitment value per CLAUDE.md's current supply-bottleneck priority), then `/sell`, then `/shop`/`/help`. Draft copy angle: lead with the "one founding seller per country" mechanic and the culture/heritage differentiator (not "handmade"), never invent numbers not already backed by code (see the copy-honesty comment at the top of `app/sell/page.tsx`).
2. **[HIGH] Add Organization + WebSite JSON-LD schema.** Confirmed via repo-wide grep: zero `application/ld+json` or `schema.org` references anywhere in `app/` or `components/`. Single-file, additive fix: add a JSON-LD `<script>` block in `app/layout.tsx` with `@type: Organization` (name, url, logo) and `@type: WebSite` (name, url, potentialAction search). This is the single lowest-risk, highest-leverage structured-data fix available and directly improves AI-visibility/answer-engine readiness, which matters more than usual here since the site has no organic search history pre-launch. Follow with `BreadcrumbList` on `/shop` and product pages, and `FAQPage` on `/help` once its existing FAQ content is inventoried, once the catalogue has real products behind it.
